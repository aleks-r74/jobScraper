What is it?

This web app was built as a personal AI Assistant to showcase my development skills.

What problem does it solve?

It saves my time by autonomously parsing job postings and making human-like decisions about them, thanks to an LLM (Large Language Model).

What technology does it use?

The emphasis in this project is on back-end technology.

Languages Java SE, JPQL
Framework Spring Boot
Starters Spring MVC and REST, Spring Data JPA, Spring Security, Thymeleaf
Automation Selenium WebDriver
Database MySQL
Front-end Bootstrap, HTML, CSS, Flex
LLM OpenAI, Ollama
The app comprises two independent systems

JobScraper

Runs on my local server.

  • - Extracts job postings from LinkedIn and stores them in the database.
  • - Processes found jobs using Large Language Model.
  • - Makes a human-like judgment on whether the job is worth my attention.

WebFace

Runs in the cloud.

  • - Renders results for the front-end according to user authority.
  • - Communicates with JobParser via REST API, enabling seamless data exchange.

The division of the App into two independent systems helps reduce costs associated with hosting and make it more resilient. The lightweight WebFace doesn't require powerful cloud server and provides continuous availability 24 / 7.

Main perks of the App
  • - Ability to use locally installed LLM
  • - Independent models for summarization and decision making
  • - Multi-threading parsing for faster processing

In the current setup, the lightweight Gemma:2b model on my local server performs job summarization. While this model is sufficient for text summarization, it lacks logical reasoning capabilities. To address this limitation, the final decision on whether a job aligns with my skills and experience is outsourced to OpenAI's GPT-3.5-turbo model. This combination of large language models optimally utilizes my server's hardware capabilities and reduces costs, striking a balance between performance and efficiency.
